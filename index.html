Methodology & Definitions Guide

System Overview

User Inputs→Feature Extraction→Multi-source Retrieval→Scoring & Re-ranking→Explainable Output

The app has three engines: Journal Search, Literature Search, and Literature Network Search. All scores are relevance indicators derived from text matching and citation graph signals; they are not acceptance probabilities.

Journal Search: Methods & Math

Manuscript + References→Category Memberships→OpenAlex Source Retrieval→Curated+Live Merge→Diversity-aware Top-N

Journal ranking blends manuscript fit, method and domain alignment, reference-derived venue signals, and SCImago-style category memberships. A diversity layer then avoids over-concentration in one publisher or field.

Category membership model (SCImago-style):

rawk = Σt∈x wk,t · 1(t∈ck),   pk = exp(rawk/T) / Σm exp(rawm/T)

prior(j) = Σk pk · affinity(j,k),   scorej = basej + λ · prior(j) + η · refj + δ · domainj

effectiveRankj = scorej - α · Npublisher(j) - β · Nfield(j)

Literature Search: Methods & Math

Idea + Seed Citations→Multi-API Retrieval→Deduplication→Re-ranking→Primary + Deep Dive

Literature retrieval merges OpenAlex, Semantic Scholar, and Crossref, plus seed-citation citing graphs. Results are deduplicated, then reranked with theme, overlap, phrase, and penalty logic.

Slex = a·tokenOverlap + b·phraseHits + c·seedBoost + d·themeBoost - e·topicPenalty

Sfinal = clip(ω1Slex + ω2Sprovider, 0, 100)

Ranking order: Sfinal ↓, then weightedScore ↓, then citationCount ↓.

Literature Network Search: Methods & Math

APA Seed→Resolve Canonical Work→Expand Citation Graph→Similarity Scoring→Interactive Network

A single APA seed citation is resolved to a canonical work, then expanded via citing, referenced, and related-work graph. Similarity is multi-factor, and graph distance is relative to the result set.

simi = f(titleOverlap, abstractOverlap, methodConsistency, yearProximity)

simNormi = (simi - min(sim)) / (max(sim) - min(sim) + ε)

di = dmax - (simNormi)α(dmax - dmin)

Important Notes

  • Scores are heuristic relevance indicators, not proof of fit.
  • Blank inputs reduce confidence and coverage.
  • Google Scholar has no official public API for direct in-app querying.
  • Always verify final journal scope and paper relevance manually.

How To Use (Step by Step)

1) Journal Search (Smart Match)

  1. Enter any available inputs: keywords, title/abstract, methods, references (all optional but more is better).
  2. Select quartiles/publishers, or leave broad defaults.
  3. Set number of journals to return (numeric field).
  4. Click Find Journal Matches.
  5. Review fit reasons, category memberships, and journal stats; verify final fit on journal websites.

2) Journal Catalogue

  1. Open Journal catalogue subtab.
  2. Enter a journal/topic query and optional host/publisher filter.
  3. Click Search catalogue, then Load more if needed.
  4. Use OpenAlex metrics and, when available, curated quartile/IF/CiteScore matches.

3) Literature Search

  1. Provide any subset of theme, keywords, domain, idea description, seed citations.
  2. Set the number of papers to return.
  3. Click Find Proposed Literature.
  4. Inspect paper cards (source badges, relevance, abstract, citations).
  5. Use Deep dive for citing papers and Scholar links for external validation.

4) Literature Network Search

  1. Paste one APA citation (DOI improves resolution accuracy).
  2. Click Build Literature Network.
  3. Hover/click nodes for details; use Enlarge network graph for full-page view.
  4. Read node labels as index (similarity %) and match index with the citation list below.

Best Practice Tips

  • If warning says fields are blank, add at least one more strong field to improve precision.
  • Keep seed citations high-quality and domain-representative to reduce off-topic retrieval.
  • Use results as decision support, not as final truth; always manually validate.
